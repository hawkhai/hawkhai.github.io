news/2023/10/29 21:01:50

六．在C/C++环境中调用python代码

在这种情况下，C/C++代码是主程序，按复杂度，分几个例子说明。

1.最简单情况，无参数传递

int run_simple_py()
{char const *script="def hello():\n\tprint(\"hello world!\")\nhello()";int res;Py_Initialize();res=PyRun_SimpleString(script);Py_Finalize();return res;
}

2.“编译”脚本，再引入固有库

在运行之前，“编译”一下python代码很有必要，函数是Py_CompileString，代码如下：


int run_py1()
{const char *script="def hello():\n\tprint(\"hello world!\")\nhello()";int res=0;Py_Initialize();if(!Py_IsInitialized()){res =-10;return res;}PyObject *pdicGlobal = PyDict_New();if (PyDict_GetItemString(pdicGlobal, "__builtins__") == nullptr){if (PyDict_SetItemString(pdicGlobal, "__builtins__",PyEval_GetBuiltins()) != 0){res =-4;return res;}}PyObject *pdictLocals = nullptr;PyObject* pCompile=Py_CompileString(script, "aaa",Py_file_input );if(pCompile){PyObject* pRun=PyEval_EvalCode(pCompile, pdicGlobal, pdictLocals);if(pRun)  Py_DECREF(pRun);else res=-3;Py_DECREF(pCompile);}else{res=-2;}if (res!=0 && PyErr_Occurred())  PyErr_Print();Py_XDECREF(pdictLocals);Py_CLEAR(pdicGlobal);Py_Finalize();return res;
}

C/C++中调用python代码的函数有很多种，可适应不同的情况，上例中PyRun_SimpleString是最简单的一个，无参数，但是如果调用函数中有*globals, *locals这样的参数时，实际上，此时的python解释器是“空”的，无内置库，因此诸如print这样的函数都会出现“无函数定义”的错误，解决方法就是将“__builtins__”的东西“注入”到python解释器中。

调用运行的python脚本中，引用了自己写的另一个python脚本pymodule，其代码如下：

def add(x,y):return x+y

虽然pymodule.py和C/C++运行程序在同一目录，但python的解释器却不这么认为，因此直接使用会出现“未发现pymodule”的错误，代码中采用sys.path.append将pymodule.py所在的目录添加到python的系统查找目录，解决上述问题。
