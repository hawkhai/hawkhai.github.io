Android Studio: “Error initializing ADB: Android Debug Bridge not found” 且找不到Project Structure

开发flutter时，想要访问手机上的文件，结果Android Studio中Device File Explorer中提示: “Error initializing ADB: Android Debug Bridge not found” 。

几番搜索之后，发现大家的做法都是一样

Android Studio: “Error initializing ADB: Android Debug Bridge not found”

https://stackoverflow.com/questions/47506261/android-studio-error-initializing-adb-android-debug-bridge-not-found

或者是

Flutter在Android Studio报告错误：ADB not found

https://blog.csdn.net/net_wolf/article/details/88971642

 

但是我的Android Studio中File下没有Project instructure，只有Setting和Other Setting，就连在项目根目录右击→Open Module Settings，也无法配置Project SDK！！！

 

经过我的潜心研究、外加举一反三的领悟，我删除了本地的.idea文件夹（如果是flutter开发的话把android目录下的.idea文件夹也删掉）。

删除这个文件夹也要小心翼翼的，先关掉android studio，然后删除文件夹，然后再用android studio打开项目（提示android sdk代理啥的直接忽略掉），这时候这个文件夹自己就会再出来啦，你就不用管这个文件夹了，因为这时候File下已经有Project Structure了，按照上面两篇文章说的（虽然是两篇文章，其实做法是一样的）配置SDK就好了。

发生这种情况，我估计是创建项目时，gitignore中没有忽略.idea文件夹，导致git拉取项目后用的是别人项目的.idea文件夹，这时候就会发生这种诡异的事。所以用idea开发的时候，一定要把.idea文件夹加到gitignore中哦！
