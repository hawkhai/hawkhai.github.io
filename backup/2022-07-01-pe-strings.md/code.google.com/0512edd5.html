strenc

Tool for static strings obfuscation within a C++ application

strenc()

What is strenc()?

strenc() is a custom build tool which parses user defined strings inside of a .strenc file. strenc() will then output a header file with encrypted versions of the strings and #defines which call the decryption method on the also #defined encrypted strings.

What compilers does strenc() work with?

The supplied download is for Visual Studio 2008, but it should also work with any Windows compiler which supports custom build steps.

How can I use strenc()?

  1. Go to our Downloads Page and download the latest version of strenc().
  2. Extract the .rar file.
  3. Place strenc.exe in _C:\Program Files (x86)\Microsoft Visual Studio 9.0\VC\bin_
  4. Place strenc.lib in _C:\Program Files (x86)\Microsoft Visual Studio 9.0\VC\lib_
  5. Place strenc() Files.rules in _C:\Program Files (x86)\Microsoft Visual Studio 9.0\VC\VCProjectDefaults_
  6. Right-click on your project, click "Custom Build Rules..." then select "Find Existing." Select the strenc() Files.rules file and make sure it is check in the list.
  7. Add a .strenc file to your project with the below format: STRING_ONE_DEFINE_NAME String one text STRING_TWO_DEFINE_NAME String two text
  8. Include {fileName}.h in any files where you wish to use the strings, where {fileName} is the name of the .strenc file you supplied.

What are some limitations with strenc()?

  1. Input to strenc() is parsed line by line. The first line strenc() see's will be the define name, the second line will be the actual text. For this reason, input files should always have a line-count multiple of 2.
  2. The #defines generated by strenc() are, for now, allocated dynamically and must be de-allocated by the user.
  3. The strings used in strenc() should remain short and sweet, due to static buffer sizes (this will be fixed)

What is some sample strenc() input and output?

INPUT: strings.strenc INTRO_STRING This is a test of strenc() SECOND_STRING How's it working? THIRD_STRING testing "quotes" bro.
