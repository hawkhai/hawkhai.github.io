简介

使用 NT 和 Win2K 意味着可执行文件和对象文件将多次嵌入 UNICODE 字符串，而你无法使用标准 ASCII 字符串或 grep 程序轻松查看这些字符串。 因此，我们决定滚动自己的。 字符串仅扫描传递的 UNICODE (或 ASCII) 字符串的默认长度为 3 或更多 UNICODE (或 ASCII) 字符。 请注意，它也适用于 95 Windows。

使用字符串

用法：

cmd 
strings [-a] [-f offset] [-b bytes] [-n length] [-o] [-q] [-s] [-u] <file or directory>

字符串采用文件名的通配符表达式，其他命令行参数的定义如下：

参数 说明
-a 仅限 Ascii 的搜索 (Unicode 和 Ascii 是默认)
-b 要扫描的文件字节数
-f 开始扫描的文件偏移量。
-o 文件字符串中的打印偏移量已找到
-n 默认值为 3) 的最小字符串长度 (
-s Recurse 子目录
-u 仅 Unicode 搜索 (Unicode 和 Ascii 默认为)
-nobanner 不显示启动横幅和版权消息。

若要使用字符串搜索一个或多个文件以查找特定字符串是否存在，请使用如下所示的命令：

cmd 
strings * | findstr /i TextToSearchFor
