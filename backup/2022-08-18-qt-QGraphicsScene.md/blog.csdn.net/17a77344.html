在双显示器接双显卡的场景下，Qt的QScreen()::geometry()方法存在缺陷

swift19221 于 2022-07-01 09:42:28 发布 73 收藏
文章标签： qt windows

本文链接：https://blog.csdn.net/swift19221/article/details/125552771
版权

主板有集成显卡，PCI插槽又配置了独立显卡。有两个显示器，一个接到集成显卡，另一个接到独立显卡，这样就组成了双显示器接双显卡的场景。
系统设置：桌面点击右键-->显示设置-->多显示器设置-->扩展这些显示器，这样组成了双屏扩展模式。
下面分别获取每个显示器的矩形区域。

1、用QT的方法：
void GetScreenRectsByQt(QRect& rcPrimary, QRect& rcSecond)
{
    QList<QScreen*> lsScreens = QGuiApplication::screens();
    QScreen *pPrimaryScreen = qApp->primaryScreen();
    int iCnt = 0;
    
    for (int i = 0; i < lsScreens.size(); i++) {
        QScreen* theScreen = lsScreens.at(i);
        if(theScreen == pPrimaryScreen){
            rcPrimary = theScreen->geometry() ;    // 如果不包括任务栏，则使用 availableGeometry()
            iCnt++;
        }
        else {
            rcSecond = theScreen->geometry() ;
            iCnt++;
        }
        
        if(iCnt == 2){
            break;
        }
    }
}

2、用系统API:
RECT g_rcPrimary;
RECT g_rcSecond;
BOOL CALLBACK MonitorEnumProc(HMONITOR hMonitor, HDC hdcMonitor, LPRECT lprcMonitor, LPARAM dwData)
{
    MONITORINFO monitorinfo;
    monitorinfo.cbSize = sizeof(MONITORINFO);
    if (!GetMonitorInfo(hMonitor, &monitorinfo)) {
        return TRUE;
    }

    if (monitorinfo.dwFlags == MONITORINFOF_PRIMARY) {
        g_rcPrimary = monitorinfo.rcMonitor;    // 如果不包括任务栏，则使用 monitorinfo.rcWork
    }
    else {
        g_rcSecond = monitorinfo.rcMonitor;
    }
    return TRUE;
}

void GetScreenRectsByAPI()
{
    g_rcPrimary = { 0, 0, 0, 0 };
    g_rcSecond = { 0, 0, 0, 0 };
    EnumDisplayMonitors(NULL, NULL, MonitorEnumProc, (LPARAM)nullptr);
}

比较上面1、2两个方法的结果，发现方法1获取的矩形区域是不正确的，因此用方法2来代替。
如果是单显卡接两个显示器，则没有问题。
